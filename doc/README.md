# HYRESIS : THE LAST STATEMENT

## 실행 방법
- `index.html`을 브라우저로 열면 바로 실행됩니다.

## 구현 단계(체크리스트)
- 1차: 정적 데이터 + 입력/로그 처리 + 화면 출력 완료
- 2차: 로그 다양화, 타이핑 효과, 사운드 등 연출 강화
- 3차: 문서 무결성 시각화 확장, 엔딩 연출 고도화

## 참고
- 기호 입력은 버튼 또는 키보드로 가능합니다.
- 각 Clause는 길이가 고정되어 있으며 입력 길이가 맞지 않으면 재시도를 안내합니다.

## 변경 이력

### 2026-02-08

#### 1) `mission-overlay` 단순화
- 수정 이유: 오버레이 내부 문구와 로그가 중복되어 읽기 흐름이 끊김.
- 목표: 오류 복구에 필요한 정보만 남겨 즉시 이해 가능한 입력 창으로 정리.
- 반영 내용:
  - 오버레이 헤더를 `ERROR WINDOW` 중심으로 통일.
  - `mission-log` 제거, 피드백을 `mission-tip` 한 줄로 통합.
  - 커서 깜박임/포커스 강조 등 입력 UX 중심으로 전환.

#### 2) 퍼즐 진행 UX 개선(정답 위치 고정)
- 수정 이유: 이미 맞춘 위치를 매번 다시 입력해야 해서 불필요한 반복이 발생.
- 목표: 맞춘 칸은 고정(초록)하고 남은 칸만 입력하는 방식으로 진행감 강화.
- 반영 내용:
  - `hit-correct` 슬롯 고정 상태(`lockedFormulaSlots`) 도입.
  - 수식 슬롯 시각화: 고정 칸(초록), 남은 칸(노랑).
  - 안내 문구에 `고정 N칸 / 남은 N칸` 및 `남은 1칸` 힌트 추가.

#### 3) `terminal-log`를 파일 읽기 흐름으로 재구성
- 수정 이유: 로그가 시스템 메시지 중심이라 “유언장 파일을 읽는 체험”이 약함.
- 목표: 문서를 읽다가 오류가 발생하고, 복구 후 다시 읽기가 이어지는 흐름을 명확화.
- 반영 내용:
  - 오프닝 텍스트를 `mask/doc/openning.md` 내용 기반으로 한 줄씩 출력.
  - 읽기 도중 오류 감지 시 복구 팝업 진입.
  - 복구 성공 시 임시 오류 로그를 제거하고 `[WILL]` 라인으로 읽기 재개.

#### 4) 상태 전이 안정화(시간 의존 최소화)
- 수정 이유: 지연 출력/즉시 출력이 섞이면서 로그 순서가 뒤섞일 위험이 있었음.
- 목표: 상태 전이가 항상 `읽기 -> 오류 -> 복구 -> 읽기 재개` 순서를 보장하도록 정리.
- 반영 내용:
  - 로그 출력 기본을 즉시 출력으로 고정.
  - 오프닝은 줄 단위 출력만 유지하고, 상태 전환은 논리 순서 기준으로 처리.
  - transient error 로그 관리 추가(복구 시 제거).
